MONTE CARLO INTEGRATION SMALL MULTIPLE PLOT - PLANNING DOCUMENT

PROJECT OVERVIEW

Create a series of 4 small multiple plots demonstrating Monte Carlo numerical integration at increasing sample sizes (100, 250, 1000, and 5000 points). Each plot estimates the area under a curve f(x) = √x from x=0 to x=4.

KEY CONCEPTS

Monte Carlo Integration: A method to estimate the area under a curve by:
1. Generating random points uniformly in a bounding rectangle
2. Counting how many points fall below the curve
3. Estimating: Area ≈ (Rectangle Area) × (Points Below / Total Points)

COMPONENT BREAKDOWN WITH NOUNS AND VERBS

STEP 1: COORDINATE GENERATION FUNCTION

Purpose: Create random data points for sampling

Nouns: coordinates, points, bounds, sample size, x-axis, y-axis, region, rectangle, sampling

Verbs: generate, create, random, runif (uniform random sampling)

Function: generate_coordinates()
- Input: sample size (n_samples), x bounds, y bounds
- Process: Sample uniformly random values for x and y
- Output: Tibble with x and y columns

STEP 2: FLAGGING FUNCTION

Purpose: Classify points relative to the function curve

Nouns: function value, curve, classification, flag, point location, above/below status, indicator variable

Verbs: evaluate, compare, flag, mutate, classify

Function: flag_points_relative_to_function()
- Input: Data (x, y points), function to evaluate
- Process: 
  - Calculate function_value for each x
  - Create boolean flag: is_above_function = (y > function_value)
- Output: Tibble with added is_above_function column (TRUE/FALSE)

STEP 3: INTEGRATION ESTIMATION FUNCTION

Purpose: Calculate the area estimate

Nouns: area, rectangle area, proportion, ratio, points below, total points, integration estimate

Verbs: calculate, multiply, estimate, count

Function: estimate_integration()
- Formula: Estimated Area = Rectangle Area × (Points Below / Total Points)
- Input: Flagged data, bounds
- Process:
  - Determine rectangle area from bounds
  - Count points below curve (sum of !is_above_function)
  - Multiply: area = rect_area × (points_below / total_points)
- Output: Numeric estimation value

STEP 4: VISUALIZATION FUNCTION

Purpose: Create reusable plot template

Nouns: plot, scatter points, curve line, ggplot layers, color categories, titles, subtitle, sample size

Verbs: plot, visualize, display, layer, color-code

Function: plot_monte_carlo()
- Elements:
  - Layer 1: Scatter points above curve (red, α=0.6)
  - Layer 2: Scatter points below curve (blue, α=0.6)
  - Layer 3: Function curve line (black, solid)
- Customization Verbs: label, title, subtitle, format, theme, geom_point, geom_line
- Output: ggplot object (reusable for any resolution)

STEP 5: MULTI-RESOLUTION ANALYSIS

Purpose: Execute analysis pipeline at 4 sample sizes

Nouns/Verbs Process Flow:
1. Define parameters: function, bounds, sample_sizes vector
2. Loop through each sample size
3. Generate coordinates for current size
4. Classify points (flag relative to function)
5. Calculate area estimate
6. Store results in tibble
7. Create plot for each resolution

Sample Sizes: 100 → 250 → 1000 → 5000
(Increasing resolution shows convergence to true area)

STEP 6: RESULTS ASSEMBLY

Purpose: Combine outputs into final display

Nouns: small multiples, grid, table, estimates, visualization, results summary, numerical summary

Verbs: arrange, grid, print, display, bind, summarize

Output:
1. Results Table: Shows n_samples, estimated_area, percent_below for each resolution
2. Small Multiples Grid: 2×2 arrangement showing all 4 plots
3. Numerical Annotation: True analytical value for comparison

DATA FLOW ARCHITECTURE

Define Parameters
- Function f(x)
- Bounds (x, y)
- Sample Sizes [100, 250, 1k, 5k]
         |
         ▼
For Each Sample Size:

1) GENERATE → Coordinates
2) FLAG → Above/Below Function
3) ESTIMATE → Area Calculation
4) VISUALIZE → Create Plot
5) STORE → Results & Plot
         |
         ▼
OUTPUT:
- Results Table (n, area, %)
- Small Multiples Grid (2×2)
- Comparison to True Value

CUSTOMIZATION POINTS

The script is designed to be flexible. To adapt for different functions:

Change Function:
function_to_integrate <- function(x) sqrt(x)  # modify this line

Change Bounds:
x_bounds <- c(0, 4)    # modify x range
y_bounds <- c(0, 2)    # modify y range

Change Sample Sizes:
sample_sizes <- c(100, 250, 1000, 5000)  # modify resolutions

Change True Value (for annotation):
cat("True Area:", <analytical_solution>, "\n")

EXPECTED OUTCOMES

1. Convergence: Larger sample sizes should produce more accurate estimates
2. Probability Pattern: Sample proportion × rectangle area ≈ true area
3. Visual Evidence: More points (higher resolution) → tighter, more accurate estimate
4. Reproducibility: Function-based design allows easy reuse for new curves
